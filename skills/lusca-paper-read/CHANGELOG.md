## [1.9.3] — 2026-07-17

### Changed
- 「模板使用约定」新增「有原图就不补 mermaid」：某节已嵌入原文图（架构图/结果图等）时**不再额外画 mermaid 重绘**——两份重叠内容冲突、徒增噪声；mermaid 仅用于原文无对应图、需自行归纳可视化的场景（如 1.3 相关工作脉络）。原文图与 mermaid 二选一，不并存
- 「阅读笔记结构」2.1 架构图改为"原文有图则嵌原图、不再补 mermaid；仅原文无图才 mermaid 重绘"
- 模板 2.1 占位同步：去掉"图 + mermaid"并存示范，改为嵌原图优先、注释说明无图时才用 mermaid

## [1.9.2] — 2026-07-17

### Changed
- 「模板使用约定」新增列表符号规范：并列要点用无序 `-` 逐行展开，**不要把 ①②③ 塞在同一行**（要用 ① 分点就分行写成列表项，或直接用无序）；有序列表（`1.` `2.`）仅用于真正有先后/递进/步骤关系的内容，**无递进的并列（多条贡献/假设/发现/警惕点）用无序不用有序**
- 模板 1.2 核心贡献由有序 `1. 2. 3.` 改为无序 `-` 示范，注释同步

## [1.9.1] — 2026-07-17

### Changed
- 「模板使用约定」新增 mermaid 图规范：节点 ID 用**语义化 slug**（如 `doclayout_yolo`、`mask_head`），不用 A/B/C 无意义字母；图下解读文字用节点内容（工作名+年份 / 模块名）指代，**严禁引用节点字母 ID**（"A→B→C"、"DLAFormer（I）"等）——字母只是语法占位符，对读者无意义，不应要求读者回头对照图查字母。对 1.3 相关工作流程图与 2.1 架构图均适用
- 模板 1.3 相关工作流程图的 HTML 注释同步补充该要求

## [1.9.0] — 2026-07-15

### Changed
- 输出目录改为「每篇论文一个文件夹」：`outputs/lusca-paper-read/{slug}/{timestamp}_{slug}.md` + `{slug}/imgs/`；图片须转存到 `imgs/` 并用 `![](imgs/figX_xxx.jpg)` 直接展示，**禁止只写文字路径**（断链不可视）
- 1.1 研究内容拆出「痛点」：改为 研究问题 / 痛点 / 动机·重要性 / 领域定位 四点
- 2.2 模块详解要求升级为「讲透实现原理」（输入→处理→输出 + 设计理由 + 关键参数），明确**不是列参数清单**
- 2.3 算法/伪代码要求纠正为「**原文有则转述+逐步解读**，不要凭空生成原文没有的伪代码」
- 文末出处块：笔记结尾附一行 `> 作者：… ｜ 版本：lusca-paper-read v… ｜ 出处：skills/lusca-paper-read/`（遵 CLAUDE.md「文档输出规范」），模板同步加占位
