### Added

### Changed
- 主流程 Step 2 升级：读全文时**建立图清单**（Figure 号 → 内容 → 归位节），据此下载到 `imgs/`；明确「**下载即承诺嵌入**」——进了 `imgs/` 的图正文必须引用
- 主流程 Step 5 新增**落盘图片自检**子步：跑 `scripts/check_imgs.py`，报孤儿图则补嵌或删除，不留下"下了不用"的图

- 模板 2.2 模块详解注释补"模块专属结构图（head 内部结构、数据增强流水线、注意力可视化等）也要嵌，不只靠文字"；3.1 关键结果注释补"多张结果图（主结果 + 消融 + 定性示例）各自归位，不止一张主图"
- 背景：4 份既有笔记中 3 份存在"图下载后未嵌"（orli fig3 曲线头、roap fig3a/fig3c、rt-doclayout fig3 数据增强）——子结构图 / 示例图 / 消融图最易漏嵌，故新增硬校验机制

## [1.9.8] — 2026-07-17

### Added
- 「模板使用约定」新增「mermaid / 代码块等不渲染 LaTeX 的区域里不写公式表达」：mermaid 图与 ``` 代码块/伪代码块不渲染 `$...$`/`$$...$$`（定界符被当字面字符、甚至会触发 mermaid 语法错误），这些区域表达数学/逻辑改用文字与简短符号，正式公式放正文 `$$`/`$`；若必须含公式则拆到块外正文 `$$`——针对 mermaid 节点/伪代码块里公式无法正确渲染的情况

## [1.9.6] — 2026-07-17

### Added
- 「模板使用约定」新增「数学公式用规范 LaTeX」：行内公式 `$...$`、行间公式 `$$...$$`（前后空行），**主动修复原文转写中损坏的 LaTeX**（下标/上标 `_`/`^`、希腊字母宏 `\alpha`、运算符 `\times \sum \frac{}{}`、补全定界符、还原 OCR/HTML 乱码），不要把损坏或半成品公式原样塞进笔记；模板顶部注释同步加 LaTeX 提醒——针对笔记中公式渲染错误（行内/行间混用、裸字符代替宏、转写乱码未修）

## [1.9.5] — 2026-07-17

### Changed
- 「模板使用约定·列表符号规范」强化：**真正的步骤 / 阶段 / 递进**（"分 N 步处理"、训练 / 推理流程、编码器多阶段聚合等）一律用有序列表 `1.` `2.` 逐行写，**不要 (1)(2)(3) 内联塞在一段**——针对笔记中"处理分四步：(1)... (2)... (3)... (4)..."这类分步说明未拆列表的问题；并明确"不要把 ①②③ 或 (1)(2)(3) 塞在同一段"
- 「模板使用约定」新增「段落可拆就拆，别压长段」：一段里塞了多个语义上可独立并列的要点（多个机制 / 理由 / 条件 / 子能力）时，拆成无序列表或分段，仅在语义连贯、确实构成一段完整论述时才保留整段——提高源码可读性与渲染扫读性
- 模板 2.2 模块详解：注释由 ①②③④ 改为无序列表示范，并明确"内部多步骤 / 多阶段处理流程用有序子列表分步写，不要 (1)(2)(3) 内联"；示范行由单行"输入 → 处理流程 → 输出"改为子列表结构
- 模板「整体处理流程」注释：明确为鸟瞰一段话、细节留给 2.2 分步展开，天然分若干阶段时可改有序列表逐阶段
