# Step 1: Intake — 界定会话范围与候选问题

执行本步前无需其他 prompt。

## 目标

圈定本次反省的**会话范围**，收集**候选问题**，识别**涉及的 skill**，避免漫无目的地记录。

## 必确认项

从当前会话上下文与用户消息中提取；缺失项用默认值，**不打断**：

| 项 | 说明 | 默认 |
|----|------|------|
| 会话范围 | 要反省整段会话 / 某几轮 / 某次具体失败 | 最近出现的问题 |
| 涉及 skill | 出问题的 skill 名（可多个） | 从会话中推断 |
| 候选问题 | 用户已指出的问题 + 你观察到的异常 | 逐条列出 |
| session 主题 | 用于文件命名的 slug | 取主问题关键词 |

仅在**关键信息缺失且无法从会话推断**时，合并成一条消息追问（最多 1 次）；能推断则用默认值，不打断。

## 候选问题收集

扫描本会话，列出**疑似与 skill 相关的问题**，每条记：

- **现象**：发生了什么（一句话）
- **涉及 skill**：哪个 skill
- **触发点**：哪条用户消息 / 哪个操作引发的
- **会话轮次**：便于后续追溯

**暂不做关联过滤**——过滤在下一步（`skill-linkage.md`）。此处宁可多列。

## 输出

**不在对话中输出确认**——intake 在内部完成（界定范围、收集候选、识别 skill、定 slug），直接进入 Step 2 关联判定。结果写进 reflex 文档，不单独输出。

进入 Step 2：**`Read`** `references/skill-linkage.md` 做关联判定。
