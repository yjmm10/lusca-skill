# Reflex 文档填写规范

reflex 文档 = 一个 session 的 skill 问题归档。本文件规定逐节填法与质量标准。空白模板见 `assets/reflex-template.md`。

## 逐节规范

### 元信息（frontmatter / 头部）

- **日期**：会话日期（YYYY-MM-DD）
- **session 标识**：时间戳 + 主题 slug（与文件名一致）
- **涉及 skill**：skill 名列表
- **会话范围**：反省的是哪段会话（如"第 3–7 轮"或"整段"）
- **状态**：整份文档的状态（open / confirmed / archived）

### 问题清单（核心）

每个问题一个子块，含：

| 字段 | 要求 |
|------|------|
| ID | `P1`、`P2`…（便于确认区引用） |
| 现象 | 一句话说清发生了什么 |
| 涉及 skill | skill 名 |
| 关联判定 | 为何与本 skill 相关（通过三步中的哪条；见 skill-linkage） |
| 六维 | 场景 / 根因 / 影响 / 归属 / 改进 / 验证，各一行要点 |
| 建议 | 指向 skill 文件 + 改法 + 类型 |
| 状态 | 待确认 / 已确认 / 已应用 / 已驳回 / 非skill缺陷 |

**直击要点**：六维用短句，不展开成段落；细节交给维护者读对应文件。

### 汇总

- **共性问题**：多个问题指向同一根因的，合并指出。
- **优先级排序**：按 影响 × 频率 排，阻塞 + 必现最优先。
- **非缺陷/观察项**：单独列出，不与改进建议混。

### 确认区（关键）

- 列出所有「待确认」的改进建议（表格：ID / skill / 文件 / 建议 / 类型）。
- 用户逐条标注：✅ 同意调整 / ❌ 驳回 / 🔄 待定。
- **只有用户确认后**，才由对应 skill 维护流程在 `skills/<name>/` 源执行；lusca 不执行。

## 质量标准（直击要点）

1. **每条问题可执行**：读到建议就能动手改，不需要二次分析。
2. **无无关项**：与 skill 无关的问题不混入（见 skill-linkage）。
3. **状态诚实**：未确认就是「待确认」，不预设同意。
4. **不越界**：文档里只提建议，不含"已修改 XXX"的表述。
5. **可追溯**：每条建议能回溯到会话中的具体哪轮（标注轮次或引用）。

## 命名

`reflex_{YYYYMMDDHHmmss}_{主题slug}.md`，slug 小写连字符、≤6 词。
