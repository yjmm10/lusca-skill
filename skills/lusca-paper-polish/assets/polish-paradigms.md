# 润色范式与深度指南（per-section paradigms）

## 定位

- `references/section-conventions.md` 讲**各节该说什么**（必答问题、顺序、常见毛病）；
- `references/academic-phrasebank.md` 讲**怎么说**（动词阶梯、替代表达）；
- `references/ai-tone-guardrails.md` 讲**怎么去 AI 腔 / 软化注水**；
- 本文件（assets）讲**按什么范式与深度组织**——凸什么、收什么、怎么让该节饱满且亮点突出。

润色不是无差别改通顺，而是按章节职责把**该凸的凸出来、分散的收拢、缺位的补位**。

### 先认准论文类型（用错变体是常见硬伤）

每节都区分**研究型**（research：提出新方法 + 自己的实验）与**综述型**（survey/review：梳理领域 + 指出空白 + 指引方向）两种变体。先判类型再选骨架——给综述套"我们提出 X，提升 Y%"、或给研究型论文写"本文为读者提供完整知识框架"，都是错位。

## 摘要范式

### 元骨架（共性）：定位 → 主体 → 发现 → 指向

| 段 | 功能 | 研究型填什么 | 综述型填什么 |
|----|------|--------------|--------------|
| 1 定位 | 是什么、在哪个问题/领域 | 痛点/gap + 要解决什么 | 文章类型 + 主题 + 为何此时需要 |
| 2 主体 | 做了什么 | 环境建模 + 方法（创新模块+角色） | 覆盖范围（场景/分类/模块+组织逻辑） |
| 3 发现 | 得到了什么 | 实验结论（主指标绝对值+相对增益） | 揭示空白/风险 + 填补价值 |
| 4 指向 | 向前看 | （可选）核心 take-away/意义 | 未来方向与建议 |

> 研究型灵魂 = **新方法 + 硬证据**；综述型灵魂 = **覆盖广度 + 空白定位 + 方向指引**。

### 变体 A：研究型（problem → setup → method → results）
四要素：**现有问题 / 环境建模 / 方法 / 实验结论**。最常见毛病是 setup 缺位、主指标悬空。取向：setup 立参照系、凸创新点、量化增益、领域术语提专业度、聚焦亮点不展开 limitations。

### 变体 B：综述型（定位 → 内容覆盖 → 发现空白 → 未来建议）
四要素：**研究定位 / 内容覆盖 / 发现空白 / 未来建议**。取向：凸覆盖广度与组织条理、空白定位准、建议具体可落地（不空喊"未来值得关注"）。范例（IIoT 综述）：定位"IIoT 综述"→内容"系统探讨智慧医疗/城市/交通/工业应用 + 安全隐私威胁 + 对策"→发现"现有研究缺少 ML/DL/FL/DRL 应对 IoT 安全隐私的全面梳理，本文填补"→建议"聚焦资源管理/模型设计/公平性/经济性等方向"。

### 表达参考库（借鉴非套用，空套即 AI 腔）
综述：定位词（综合综述、全面分析、系统梳理）、内容动词（探讨/分析/梳理/涵盖/归纳）、发现引出（结果显示/研究发现/揭示…填补该空白）、建议引出（建议未来聚焦/鼓励研究者借助…推动…落地）。研究型：方法引出（本文提出/设计/构造）、结果引出（在…上达到/较基线提升）；动词强度匹配证据见 `academic-phrasebank.md`。

### 摘要自检
- [ ] 按论文类型选对变体？研究型 setup 是否让主指标有参照系？方法是否说清创新点、实验是否量化？综述型覆盖是否体现广度、空白与建议是否具体？
- [ ] 领域术语一致？未在摘要展开 limitations？无注水（无据强词/凭空机制/空套模板句）？

## 引言范式

### 元骨架：漏斗式递进（宏观 → 具体）+ 显性贡献声明
从领域大背景逐步收缩到本文要解决的具体问题，末尾以条目化贡献显性收口。

### 变体 A：研究型
1. **背景与动机**：领域趋势 + 为什么这个问题重要（克制，不树稻草人）。
2. **现状与盲点**：现有工作做了什么、还缺什么（公允：精确说差异，不压低 prior work）。
3. **缺口**：把盲点凝练为本文要打的靶（1-3 个耦合缺口）。
4. **本文工作与贡献**：条目化贡献（每条 = 面向哪个缺口 + 新方法 + 量化证据）。
5. （可选）**文章组织**：逐节导航。

**常见毛病**：①"现状"与"缺口"两节内容重复（同一盲点讲两遍）——收紧现状为纯描述、把凝练留给缺口，或合并；②贡献条目只罗列模块名、不说"新在哪"；③背景铺垫过长、迟迟不进 gap。

### 变体 B：综述型
1. **背景与动机**：技术演进的必然趋势（为何此时需要这篇综述）。
2. **现有综述对比**：简要指出现有综述的覆盖度与盲区（可对接 §相关工作矩阵）。
3. **本文贡献**：条目化显性列出填补的空白。
4. **文章组织**：以"导航图"逐节说明。

**表达参考**：贡献声明"通过全面总结与系统分析现有文献，本文为读者提供关于[主题]的完整知识框架，涵盖基础回顾/应用回顾/挑战与未来方向"；组织导航"如图 X，第二节介绍…，第三节探讨…，第 N 节总结全文"。

### 引言自检
- [ ] 漏斗是否真的在收缩（不是平行罗列）？现状与缺口有无重复？
- [ ] 贡献是否条目化、每条点明"新在哪 + 证据"？背景有没有过长铺垫？
- [ ] 有无空套（"激发了本研究""完整知识框架"等不承载信息的套话慎用）？

## 相关工作范式

### 元骨架：按线索组织 + 差异化定位，导出本文不可替代性

### 变体 A：研究型
- 按**技术线索**分组（2-4 组），逐组**叙事式点评**差异（不必上大矩阵）；末尾一句"本文与上述的区别在于…"。
- 小结可配精简对比表（只列与本文差异最相关的 2-3 维），不追求穷尽。

### 变体 B：综述型（对比矩阵 + 精准批判）
1. **构建对比矩阵**：设多维评估准则（如 C1…Ck），表头 `| 参考文献 | 年份 | 类型 | C1 | … | Ck | 备注 |`。
2. **符号化标注**：用 ✓/✗/∂/μ 等标识每篇对各维度的覆盖度。
3. **备注栏精准批判**：针对未覆盖或浅尝辄止的维度具体批评。

**差异化点评句型（公允、精准、不树稻草人）**：
- 覆盖面窄："对于 C6，应用覆盖范围狭窄且仅局限于语义场景。"
- 缺乏深度："对于 C5，仅简要提及优化而缺乏方法论深度。"
- 技术滞后："对于 C3，关注 GAI 但排除了 Mamba、SSM 等较新架构。"
- 缺方案："对于 C5，提及了优化挑战但未提出具体解决方案。"
- 本文定位："本文提供一份全面且最新的综述，包含深入讨论、分析、挑战及未来方向。"

### 相关工作自检 + 公允护栏
- [ ] 是否按线索组织（而非按论文流水账）？差异化是否精准到具体维度？
- [ ] **公允**：批判是否针对"没做什么/做得多浅"，而非贬低"做得很差"？有没有把 prior work 压成稻草人来衬托本文？
- [ ] 矩阵准则是否与本文贡献相关（不为列而列）？

## 方法 / 框架范式

### 元骨架：全局架构 → 组件解构 → 机制讲透

### 变体 A：研究型（本文方法）
1. **整体流程一段通讲**：模块组成 + 数据怎么流转（让读者先有全景）。
2. **逐模块讲透原理**：每个模块的输入→处理→输出 + 为什么这么设计 + 关键参数——**讲透实现原理，不是列参数清单**。
3. **伪代码/算法**：原文有则转述 + 逐步解读关键步骤；原文无则 n/a，不凭空生成。

**常见毛病**：①只列模块名/参数表，不讲"为什么这么设计、怎么起作用"；②有架构图又重复画 mermaid（二选一）；③伪代码凭空生成原文没有的。

### 变体 B：综述型（架构解构 + 使能技术分层）
1. **框架全局提出**：命名 + 核心目标 + 组成（如"MEI4LLM 框架，支持边缘部署 LLM，由 AI 原生架构/参数共享缓存/分布式训练/分布式推理组成"）。
2. **基础架构与分类**：底层模型类别（Transformer/Diffusion/Mamba；LLM/LVM/LMM；预训练/微调/对齐）。
3. **关键使能技术分模块**：逐模块阐述实现路径（如"紧凑模型与压缩/能量感知计算/鲁棒连接/知识表示"）。
4. **具体算法机制**：点名代表性机制（如"采用跨模态注意力图识别高语义图像区域"）。

### 方法自检
- [ ] 有没有"整体流程一段通讲"先给全景？逐模块是否讲透原理（而非列清单）？
- [ ] 有原图是否避免重复 mermaid？伪代码是否只转述原文已有的？
- [ ] 综述型：架构/分类/使能技术层次是否清晰？

## 实验 / 评估范式

### 元骨架：设置 → 结果 → 分析（隔离效应/反思）

### 变体 A：研究型（自己的实验）
1. **实验设置与评估协议**：数据/环境/基线/指标/训练协议（服务于可复现）。
2. **总体性能**：主指标绝对值 + 相对增益（配表/图）。
3. **消融/析因隔离效应**：拆解各模块/组件的贡献（凸创新有效）。
4. **稳健性与局限**：不同条件下的表现 + 诚实局限。

**常见毛病**：①只给结论不给支撑数字；②析因/消融铺陈过细挤占主结果；③报喜不报忧（关键不利结果藏起来）。

### 变体 B：综述型（归纳别人的实验）
1. **按场景分类归纳**：按应用层级/任务类型分小节。
2. **实验目标与算法汇总表**：`| 场景 | 技术 | 参考文献 | 目标/性能 |`。
3. **定量结果提炼**：提取关键性能提升数据（如"碰撞率降低 81.04%、感知人类似度提高 50%"）。
4. **局限性反思**：指出实验设置的缺陷（如"缺乏对原始数据的直接访问""未在真实云基础设施实测"）。

### 实验自检
- [ ] 研究型：主结果是否给量化（绝对值+相对）？消融是否凸出创新有效？有无报喜不报忧？
- [ ] 综述型：场景分类是否清晰？定量提炼是否标注来源？局限反思是否具体？
- [ ] 数字是否与正文/表格一致（不矛盾）？

## 结论范式

### 元骨架：回顾 → 意义/局限 → 指向

### 变体 A：研究型
1. **本文贡献回顾**：凝练核心方法 + 关键结果（不引入新内容）。
2. **局限**：诚实点出边界。
3. **未来工作**：基于真实局限/机会的下一步。

**常见毛病**：①把结论写成 Discussion 的复述；②引入新实验/新论断；③泛泛"未来大有可为"。

### 变体 B：综述型（凝练 + 前瞻指引）
1. **核心回顾与教训提炼**：概括全文核心机制与启示（"核心理念是[一句话]，以提升[效率/鲁棒性]"）。
2. **系统级挑战与部署策略**：落地阻碍（资源受限、安全隐私等）。
3. **未来方向**：新兴技术路径的具体指引（"建议未来聚焦[数据/可解释性/部署]，借助[智能体化等]推动[模型]在[下一代网络]中的可扩展与可信赖部署"）。

### 结论自检
- [ ] 是否凝练（不复述正文、不引入新内容）？
- [ ] 局限是否诚实？未来方向是否基于真实局限/机会（非空喊）？

## 边界重申（范式不凌驾于诚信之上）

范式指导**方向与深度**，但下述准则优先级更高，冲突时以它们为准：

- **忠实本意 / 绝不臆造**：补内容只用原稿已有要素；原文没有的数据、机制、引用一律不加。
- **校准**：措辞强度匹配证据；"显著/大幅/全面"等词要有实质撑。
- **公允**：不压低 prior work 来衬托本文（相关工作尤其）。
- **去 AI 腔**：所有模板句/高频词**借鉴非套用**；不承载信息的套话一律删（"激发了本研究""完整知识框架""大有可为"等空壳慎用或删）。
- **论文类型别选错**：研究型/综述型变体不可混用。

"突出亮点、避免短板"是**组织策略**（该凸的凸、limitations 不在摘要展开），**不是**授权注水或隐瞒。粉饰出来的章节比克制的章节伤害更大。
