# 结论范式

## 本节专属审稿维度

**未来方向可落地（三要素）+ 首尾闭环回链引言 Gap**：未来方向须给"问题 + 使能技术 + 预期目标"
的技术路径，忌假大空；结论的开放问题须与引言指出的 Gap 首尾呼应、形成闭环。

## 组织范式：回顾 → 意义/局限 → 指向

### 变体 A：研究型
1. **本文贡献回顾**：凝练核心方法 + 关键结果（不引入新内容）。
2. **局限**：诚实点出边界。
3. **未来工作**：基于真实局限/机会的下一步。

**常见毛病**：①把结论写成 Discussion 的复述；②引入新实验/新论断；③泛泛"未来大有可为"。

### 变体 B：综述型（凝练 + 前瞻指引）
1. **核心回顾与教训提炼**：概括全文核心机制与启示（"核心理念是[一句话]，以提升[效率/鲁棒性]"）。
2. **系统级挑战与部署策略**：落地阻碍（资源受限、安全隐私等）。
3. **未来方向**：新兴技术路径的具体指引（"建议未来聚焦[数据/可解释性/部署]，借助[智能体化等]推动[模型]在[下一代网络]中的可扩展与可信赖部署"）。

## 审稿人避坑（隐性严谨性事故）

- [通用] **未来方向假大空**：写"未来需进一步研究安全问题" → 审稿人批正确的废话。须给**技术
  路径三要素**：指出问题 + 给出使能技术 + 预期目标（例："借助 Agentic system 推动 LAMs 在
  动态环境适配""建议聚焦高质量数据构建/可解释性提升/受限部署优化"）。
- [通用] **与引言 Gap 不闭环**：引言说前人缺 Mamba 架构关注，结论却不提 Mamba 落地 → 首尾
  断裂、逻辑不自洽。**结论的开放问题须回链引言中指出的空白维度**（引言矩阵标了哪些 C 维度
  缺，结论就对应提哪些方向），形成闭环。
- [综述] **挑战表述空泛**：写"存在安全和隐私挑战" → 须具体（如"LAMs 因通信理论融合不足、
  决策依赖统计相关，难以解决复杂通信问题"），引领域原文佐证。
- [通用] **引入新内容**：结论里塞新实验/新论断/新引用 → 结论只凝练回顾，不引入新内容。
- [通用] **泛泛"未来大有可为"**：空壳套话 → 删，换成具体可落地的技术路径。

## 结论自检
- [ ] 是否凝练（不复述正文、不引入新内容/新引用）？
- [ ] **未来方向三要素**：每条是否含"问题 + 使能技术 + 预期目标"（非假大空）？
- [ ] **首尾闭环**：结论的开放问题是否回链引言指出的 Gap（引言矩阵标的空白维度）？
- [ ] 局限是否诚实、具体？挑战表述是否引领域原文佐证（非空泛）？
