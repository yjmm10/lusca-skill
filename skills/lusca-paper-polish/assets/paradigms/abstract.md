# 摘要范式

## 本节专属审稿维度

**证据锚点 / 可证伪性**：摘要是论文的"微缩法庭判决书"，每一项论断（"现有缺 X""本文填补空白"）
都必须在正文有对应"案卷"（对比矩阵 / 分类树 / 实验表）支撑。声称"全面"却在正文未建维度矩阵，
即虚假声明、学术营销。

## 组织范式

### 元骨架（共性）：定位 → 主体 → 发现 → 指向

| 段 | 功能 | 研究型填什么 | 综述型填什么 |
|----|------|--------------|--------------|
| 1 定位 | 是什么、在哪个问题/领域 | 痛点/gap + 要解决什么 | 文章类型 + 主题 + 为何此时需要 |
| 2 主体 | 做了什么 | 环境建模 + 方法（创新模块+角色） | 覆盖范围（场景/分类/模块+组织逻辑） |
| 3 发现 | 得到了什么 | 实验结论（主指标绝对值+相对增益） | 揭示空白/风险 + 填补价值 |
| 4 指向 | 向前看 | （可选）核心 take-away/意义 | 未来方向与建议 |

> 研究型灵魂 = **新方法 + 硬证据**；综述型灵魂 = **覆盖广度 + 空白定位 + 方向指引**。

### 变体 A：研究型（problem → setup → method → results）
四要素：**现有问题 / 环境建模 / 方法 / 实验结论**。最常见毛病是 setup 缺位、主指标悬空。取向：setup 立参照系、凸创新点、量化增益、领域术语提专业度、聚焦亮点不展开 limitations。

### 变体 B：综述型（定位 → 内容覆盖 → 发现空白 → 未来建议）
四要素：**研究定位 / 内容覆盖 / 发现空白 / 未来建议**。取向：凸覆盖广度与组织条理、空白定位准、建议具体可落地（不空喊"未来值得关注"）。范例（IIoT 综述）：定位"IIoT 综述"→内容"系统探讨智慧医疗/城市/交通/工业应用 + 安全隐私威胁 + 对策"→发现"现有研究缺少 ML/DL/FL/DRL 应对 IoT 安全隐私的全面梳理，本文填补"→建议"聚焦资源管理/模型设计/公平性/经济性等方向"。

### 表达参考库（借鉴非套用，空套即 AI 腔）
综述：定位词（综合综述、全面分析、系统梳理）、内容动词（探讨/分析/梳理/涵盖/归纳）、发现引出（结果显示/研究发现/揭示…填补该空白）、建议引出（建议未来聚焦/鼓励研究者借助…推动…落地）。研究型：方法引出（本文提出/设计/构造）、结果引出（在…上达到/较基线提升）；动词强度匹配证据见 `../../references/academic-phrasebank.md`。

## 审稿人避坑（隐性严谨性事故）

- [通用] **声称无正文支撑**：写"本文全面分析/填补空白"却未在正文建维度矩阵或分类树 → 审稿人
  要求证据，找不到即判虚假声明。摘要的每个"全面/填补/首次"都要能在正文定位到对应矩阵/表/树。
- [综述] **空白表述模糊**：只说"现有研究不足"，不说**哪个维度**缺 → 须精确到维度（"现有综述
  主要聚焦 LLM，对 world model 关注不足"），与正文对比矩阵的 C1-Ck 对齐。
- [研究型] **主指标悬空**：只说"效果很好"、不给绝对值或相对增益 → setup 缺位时尤其要补参照系；
  无数据则标"（建议补主指标）"，不编。
- [通用] **摘要展开 limitations**：把局限写进摘要冲淡亮点 → limitations 移到 Discussion/结论，
  摘要聚焦 take-away。

## 摘要自检
- [ ] 按论文类型选对变体？研究型 setup 是否让主指标有参照系？方法是否说清创新点、实验是否量化？综述型覆盖是否体现广度、空白与建议是否具体？
- [ ] 每个"全面/填补/首次"都能在正文定位到对应矩阵/表/树（证据锚点）？
- [ ] 领域术语一致？未在摘要展开 limitations？无注水（无据强词/凭空机制/空套模板句）？
