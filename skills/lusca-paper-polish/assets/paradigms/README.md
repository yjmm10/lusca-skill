# 润色范式索引（per-section paradigms）

## 定位（与 references 的分工）

- `references/section-conventions.md` 讲**各节该说什么**（必答问题、顺序、常见毛病）；
- `references/academic-phrasebank.md` 讲**怎么说**（动词阶梯、替代表达）；
- `references/ai-tone-guardrails.md` 讲**怎么去 AI 腔 / 软化注水**；
- 本目录（`assets/paradigms/`）讲**按什么范式与深度组织 + 审稿人会怎么挑刺**——每节一个文件，
  按需 Read，不必全读。

润色不是无差别改通顺，而是按章节职责把**该凸的凸出来、分散的收拢、缺位的补位**，同时守住
**审稿人视角的严谨性底线**（哪些是会被挑刺的隐性硬伤）。

## 先认准论文类型（用错变体是常见硬伤）

每节都区分**研究型**（research：提出新方法 + 自己的实验）与**综述型**（survey/review：梳理领域 + 指出空白 + 指引方向）两种变体。先判类型再选骨架——给综述套"我们提出 X，提升 Y%"、或给研究型写"本文为读者提供完整知识框架"，都是错位。

## 各节专属审稿维度速查

润色某节时，除通用的 faithfulness / 校准 / 去 AI 腔外，按该节专属维度重点核查：

| 节 | Read 文件 | 专属审稿维度 |
|---|---|---|
| 摘要 | `abstract.md` | 证据锚点 / 可证伪性（声称"全面/填补空白"须有正文矩阵撑） |
| 引言 | `introduction.md` | 贡献可验证 + 差异化坐标 + 导航图-正文对应 |
| 相关工作 | `related-work.md` | 符号-备注一致 + 建设性否定（缺陷定位+本文补位） |
| 方法/框架 | `methods.md` | 图自解释 + 分类正交完备 + 缩写首现全称 + 物理约束不漏 |
| 实验/评估 | `experiments.md` | 控制变量 + 数字带前提 + 局限真诚（Limitations 专列） |
| 结论 | `conclusion.md` | 未来方向三要素（问题+使能技术+目标）+ 首尾闭环回链引言 Gap |

## 通篇横向维度（跨节核查）

润色整篇（非单节）时，除各节专属维度外，还需核查两条**贯穿全篇**的要求：

| 维度 | 要求 | 典型违反 |
|---|---|---|
| **图表互补不重复** | 图与表分工——图给可视化 / 架构 / 权衡空间，表给精确数值 / 多维结构化对比；同主题信息**不重复**呈现。有架构图就不另列同主题小表；表只用于正文难以简洁表达的结构化信息（实验数据、对比矩阵等） | §4.1 已有系统架构图（图 2，caption 含三层模块与职责）又另列 2 行“模块层次与职责”表（信息是图子集）→ 删表，由图 caption 承载 |
| **章节衔接（过渡自然、逻辑承接）** | 节与节之间有**承接逻辑**：上节末点出“为何进入下节”、下节开头回扣上节结论；读者顺读感到层层递进，而非各管各的硬拼 | 引言三缺口 → 相关工作按缺口线索组织并回扣；§5 各小节末给过渡句（如 §5.2 末“由 §5.3 析因分解给出”引出 §5.3）；结论回链引言 gap 形成首尾闭环 |

> 这两条不属任何单节，但整篇润色时**必查**——长文润色易“逐节精修、跨节失联”。

## 总信条（顶刊审稿人的连环追问）

> 用矩阵代替形容词，用符号代替模糊，用局限代替吹嘘，用闭环代替发散。

每一句话都要经得起三连追问：**为什么这么说 / 证据在哪 / 那又怎样**。表格、图表、符号不是装饰，
而是构建自证逻辑链的承重墙。

## 边界重申（范式不凌驾于诚信之上）

范式与审稿维度都指导**方向与深度**，但下述准则优先级更高，冲突时以它们为准：

- **忠实本意 / 绝不臆造**：补内容只用原稿已有要素；原文没有的数据、机制、引用一律不加。
- **校准**：措辞强度匹配证据；"显著/大幅/全面"等词要有实质撑。
- **公允**：不压低 prior work 来衬托本文（相关工作尤其）。
- **去 AI 腔**：所有模板句/高频词**借鉴非套用**；不承载信息的套话一律删。
- **论文类型别选错**：研究型/综述型变体不可混用。

"突出亮点、避免短板"是**组织策略**（该凸的凸、limitations 不在摘要展开），**不是**授权注水或隐瞒。
