# 方法 / 框架范式

## 本节专属审稿维度

**图自解释 + 分类正交完备 + 缩写首现全称 + 物理约束不漏**：总体架构图须能脱离正文独立看懂，
区块名与正文小节标题一一对应；分类须正交（不重叠）且完备（全覆盖）；缩写首现给全称、RL 框架
引形式化定义；不漏领域物理约束（如 UAV 气动、能量）。

## 组织范式：全局架构 → 组件解构 → 机制讲透

### 变体 A：研究型（本文方法）
1. **整体流程一段通讲**：模块组成 + 数据怎么流转（让读者先有全景）。
2. **逐模块讲透原理**：每个模块的输入→处理→输出 + 为什么这么设计 + 关键参数——**讲透实现原理，不是列参数清单**。
3. **伪代码/算法**：原文有则转述 + 逐步解读关键步骤；原文无则 n/a，不凭空生成。

**常见毛病**：①只列模块名/参数表，不讲"为什么这么设计、怎么起作用"；②有架构图又重复画 mermaid、或另列同主题小表（图表互补不重复，见 `README.md` 通篇横向维度）；③伪代码凭空生成原文没有的。

### 变体 B：综述型（架构解构 + 使能技术分层）
1. **框架全局提出**：命名 + 核心目标 + 组成（如"MEI4LLM 框架，支持边缘部署 LLM，由 AI 原生架构/参数共享缓存/分布式训练/分布式推理组成"）。
2. **基础架构与分类**：底层模型类别（Transformer/Diffusion/Mamba；LLM/LVM/LMM；预训练/微调/对齐）。
3. **关键使能技术分模块**：逐模块阐述实现路径（如"紧凑模型与压缩/能量感知计算/鲁棒连接/知识表示"）。
4. **具体算法机制**：点名代表性机制（如"采用跨模态注意力图识别高语义图像区域"）。

## 审稿人避坑（隐性严谨性事故）

- [通用] **架构图不能自解释**：总体架构图（Overall organization）须能脱离正文独立看懂；图中
  每个区块命名须与正文小节标题**一一对应**，审稿人对照图表查框架有无遗漏模块（漏了即"空中楼阁"）。
- [综述] **分类不正交不完备**：分类须**正交**（不重叠）且**完备**（全覆盖）——审稿人会刁难
  "你这个分类下，XX 模型属于哪一类？不属于就是缺陷"。主动覆盖最新架构（如把大模型分为
  LLM/LVM/LMM 时勿漏 world model，否则被批"limited model coverage"）。
- [通用] **缩写首现无全称**：任何缩写（SSM/GAI/MEI/MARL/PPO…）首次出现须给全称。
- [通用] **RL/形式化框架无定义**：涉及强化学习等框架（MARL/MADDPG/PPO），描述其状态/动作/
  奖励时即使综述也应给形式化定义的引用，不能仅文字泛泛而谈。
- [通用] **漏物理/领域约束**：框架须符合领域物理约束（如 UAV 须 comply with aerodynamics and
  energy availability；漏掉气动/能量控制即空中楼阁）。
- [通用] **有架构图又重复画 mermaid**：二选一，不重复。
