# 引言范式

## 本节专属审稿维度

**贡献可验证性 + 差异化坐标 + 导航图-正文对应**：贡献条目必须具体到后文可逐一验证；综述须建
差异化坐标轴定位与已有综述的原子级差异；导航图/组织表的小节标题须与正文一一对应。

## 组织范式：漏斗式递进（宏观 → 具体）+ 显性贡献声明

从领域大背景逐步收缩到本文要解决的具体问题，末尾以条目化贡献显性收口。

### 变体 A：研究型
1. **背景与动机**：领域趋势 + 为什么这个问题重要（克制，不树稻草人）。
2. **现状与盲点**：现有工作做了什么、还缺什么（公允：精确说差异，不压低 prior work）。
3. **缺口**：把盲点凝练为本文要打的靶（1-3 个耦合缺口）。
4. **本文工作与贡献**：条目化贡献（每条 = 面向哪个缺口 + 新方法 + 量化证据）。
5. （可选）**文章组织**：逐节导航。

**常见毛病**：①"现状"与"缺口"两节内容重复（同一盲点讲两遍）——收紧现状为纯描述、把凝练留给缺口，或合并；②贡献条目只罗列模块名、不说"新在哪"；③背景铺垫过长、迟迟不进 gap。

### 变体 B：综述型
1. **背景与动机**：技术演进的必然趋势（为何此时需要这篇综述）。
2. **现有综述对比**：简要指出现有综述的覆盖度与盲区（可对接 §相关工作矩阵）。
3. **本文贡献**：条目化显性列出填补的空白。
4. **文章组织**：以"导航图"逐节说明。

**表达参考**：贡献声明"通过全面总结与系统分析现有文献，本文为读者提供关于[主题]的完整知识框架，涵盖基础回顾/应用回顾/挑战与未来方向"；组织导航"如图 X，第二节介绍…，第三节探讨…，第 N 节总结全文"。

## 审稿人避坑（隐性严谨性事故）

- [通用] **贡献空泛不可验证**：写"本文全面分析了…"这种空话 → 审稿人在后文逐一寻找对应详述，
  找不到即判虚假声明。每条贡献须具体到"面向哪个缺口 + 新方法/新分类 + 可在后文定位的章节/
  组件"（如"提出 MEI4LLM 框架，由 AI 原生架构/参数共享缓存/分布式训练/分布式推理组成"——
  后文必须有这四个组件的详述）。
- [综述] **缺差异化坐标（伪综述）**：仅列举已有综述、无矩阵式对比 → 审稿人认为综述缺乏存在
  价值。Related survey work 须建 C1-Ck 维度坐标精准定位差异（如"Most existing surveys
  primarily focus on LLMs, paying insufficient attention to world models"）。
- [通用] **导航图与正文不对应**：Taxonomy 图/组织表的小节标题须与正文一一对应；审稿人对照
  图表查框架有无遗漏模块，对不上即判组织混乱。
- [通用] **背景铺垫过长**：迟迟不进 gap，浪费版面 → 收紧背景，尽快凝练到本文要打的靶。

## 引言自检
- [ ] 漏斗是否真的在收缩（不是平行罗列）？现状与缺口有无重复？
- [ ] 贡献是否条目化、每条点明"新在哪 + 证据"，且都能在后文定位到对应详述（可验证）？
- [ ] 综述型有无差异化坐标矩阵？导航图/组织表的小节标题与正文一一对应？
- [ ] 有无空套（"激发了本研究""完整知识框架"等不承载信息的套话慎用）？
