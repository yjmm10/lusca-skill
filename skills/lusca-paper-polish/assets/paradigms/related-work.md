# 相关工作范式

## 本节专属审稿维度

**符号-备注一致 + 建设性否定**：综述型对比矩阵的符号须自洽且与备注栏文字一致（标了"部分
覆盖"就必须解释缺了什么）；批判须带建设性——"指出缺陷 + 暗示本文补全方案"，不纯否定。

## 组织范式：按线索组织 + 差异化定位，导出本文不可替代性

### 变体 A：研究型
- 按**技术线索**分组（2-4 组），逐组**叙事式点评**差异（不必上大矩阵）；末尾一句"本文与上述的区别在于…"。
- 小结可配精简对比表（只列与本文差异最相关的 2-3 维），不追求穷尽。

### 变体 B：综述型（对比矩阵 + 精准批判）
1. **构建对比矩阵**：设多维评估准则（如 C1…Ck），表头 `| 参考文献 | 年份 | 类型 | C1 | … | Ck | 备注 |`。
2. **符号化标注**：用 ✓/✗/∂/μ 等标识每篇对各维度的覆盖度。
3. **备注栏精准批判**：针对未覆盖或浅尝辄止的维度具体批评。

**差异化点评句型（公允、精准、不树稻草人）**：
- 覆盖面窄："对于 C6，应用覆盖范围狭窄且仅局限于语义场景。"
- 缺乏深度："对于 C5，仅简要提及优化而缺乏方法论深度。"
- 技术滞后："对于 C3，关注 GAI 但排除了 Mamba、SSM 等较新架构。"
- 缺方案："对于 C5，提及了优化挑战但未提出具体解决方案。"
- 本文定位："本文提供一份全面且最新的综述，包含深入讨论、分析、挑战及未来方向。"

## 审稿人避坑（隐性严谨性事故）

- [综述] **符号无图例 / 符号与备注不一致**：用了 μ/∂/ω/∇ 等符号却无图例说明，或标了"部分
  覆盖"但备注栏没解释缺了什么 → 审稿人判学术不严谨。表下必须有清晰图例；**每个非完全覆盖
  的符号都必须有备注栏文字交代缺口**。
- [综述] **纯否定批判**：只说"前人不好/做得很差" → 显狭隘、不客观。顶级写法是**建设性否定**：
  "缺陷定位 + 本文补位"双语句型——说"前人提及优化挑战但未提具体方案"（备注），潜台词即本文
  提出了具体优化方案。保持学术客观，不用过负面词汇。
- [通用] **树稻草人**：把 prior work 压低/曲解来衬托本文 → 审稿人看得出来且不诚实。批判须针对
  "没做什么/做得多浅"（事实），而非贬低"做得很差"（价值判断）。
- [综述] **矩阵准则与本文贡献无关**：为列而列、维度堆砌 → 准则须与本文填补的空白对应，能在
  末段导出"本文不可替代性"。

## 相关工作自检 + 公允护栏
- [ ] 是否按线索组织（而非按论文流水账）？差异化是否精准到具体维度？
- [ ] **符号一致**：每个非完全覆盖的符号都有备注栏文字交代缺口？图例清晰？
- [ ] **建设性否定**：批判是否"缺陷定位 + 本文补位"，而非纯否定？
- [ ] **公允**：有没有把 prior work 压成稻草人来衬托本文？
- [ ] 矩阵准则是否与本文贡献相关（不为列而列）？
